# 공동주택 위험성평가 SaaS — 구현 계획

엄청난 양의 명세입니다. 1차 출시 범위를 정확히 맞추되, **단일 응답에 모두 구현하기엔 코드 규모가 크기 때문에** 다음 순서로 단계별 진행하는 것을 제안드립니다. 먼저 이 계획을 승인해주시면 즉시 1단계부터 작업을 시작하겠습니다.

## 기술 스택 (확정)
- TanStack Start v1 (React 19) + TypeScript + Tailwind v4
- Lovable Cloud (Supabase: PostgreSQL + Auth + Storage + RLS)
- Pretendard CDN, Lucide React, Recharts
- 디자인 토큰: `src/styles.css` oklch 변수 (--color-primary #1E40AF 등)

## 구현 단계

### 1단계 — 인프라 & 디자인 시스템 & 인증
- Lovable Cloud 활성화
- DB 스키마 전체 생성 (users/companies/complexes/complex_members/assessments/hazards/measures/participants/signatures/near_miss) + RLS 정책
- 시드 데이터 50개 표준 유해·위험요인 (별도 `hazard_library` 테이블)
- Pretendard 폰트 + 색상 토큰 + 모바일 우선 반응형 레이아웃
- 회원가입/로그인 페이지 (이메일+PW, 카카오 버튼 UI만, 직책/소속 입력)
- 가입 시 트리거로 샘플 단지 1개 + 샘플 평가 3건 + 참여자 4명 자동 생성
- 사이드바(데스크탑) + 하단 탭바(모바일) 셸

### 2단계 — 단지용 핵심 플로우 (페이지 2~6)
- 페이지 2: 대시보드 (KPI 카드 4개, 도넛 차트, 최근 평가 5개)
- 페이지 3: 사전준비 6단계 마법사 (5종 평가방법 카드 + 작업명 기반 ★추천 매핑)
- 페이지 4: 유해·위험요인 파악 (9개 카테고리 드롭다운, 시드 50개 체크박스, 사진 업로드, 아차사고)
- 페이지 5: 위험성 결정 — **5종 UI 모두 분기** (3단계 라디오 / 5단계 / 5×5 매트릭스 / 체크리스트 / OPS)
- 페이지 6: 감소대책 (4단계 우선순위 드롭다운, 잔여 위험성 재평가, 증빙 업로드)

### 3단계 — 공유/기록 + 본사 콘솔 (페이지 7~9 + 본사 3개)
- 페이지 7: 협의·공유 (QR/알림톡/링크 버튼 UI, 모바일 확인 페이지, **캔버스 전자서명**, IP/UA/시각 자동 기록)
- 페이지 8: 기록·보존 (필터·검색·정렬, PDF/엑셀 버튼은 "준비 중")
- 페이지 9: 설정 (단지 정보, 프로필, 표준 라이브러리 커스터마이징)
- 본사 1~3: 본사 대시보드 / 단지 목록 / 표준화 환산 통계 (3단계→5단계, 체크리스트 보완 수→5단계, OPS 요인 수→5단계 환산 로직)

### 후속 표시("준비 중"):
카카오 알림톡 실제 발송, PDF 출력, 결제, 자동 메일, 푸시.

## 핵심 차별점 구현 확인
1. ✅ 5종 평가방법 통합 + 작업명 기반 ★추천 매핑 테이블
2. ✅ 표준 유해·위험요인 50개 시드 (9개 카테고리)
3. ✅ 근로자 참여 자동 기록 (서명·시각·IP·UA)
4. ✅ 본사 멀티사이트 콘솔 (super_admin RLS)
5. ✅ 5단계 표준 환산 (원본 보존 + 환산값 토글)

## 한 가지 확인 요청
이 명세는 **단일 응답으로 끝내기엔 매우 큽니다** (라우트 약 15개, 컴포넌트 50+, DB 테이블 10개, 시드 50건). 안정성과 품질을 위해 **3단계로 나눠 순차 구현**하는 것을 추천드립니다. 

승인해주시면 곧바로 **1단계(인프라+인증+샘플데이터+셸)** 부터 시작하겠습니다. 한 번에 다 하길 원하시면 그것도 가능하지만, 일부 디테일이 누락되거나 컴파일 에러 위험이 있습니다.

진행 방식 선택해주세요:
- **A**: 3단계 순차 (권장, 매 단계마다 클릭 가능한 산출물)
- **B**: 한 번에 전부 (빠르지만 디테일 누락 가능)
